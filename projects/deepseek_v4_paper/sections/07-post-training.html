Alternative Reading Angle: Post-Training

Commentary

This section is the engineering heart of DeepSeek-V4 — the place where a generalist base model becomes a product-grade system. The core architectural decision is replacing mixed RL with On-Policy Distillation (OPD): instead of trying to train one model on all domains simultaneously (where gradient signals from different tasks interfere), they train a dozen+ independent specialists, then merge them into a single student via logits-level KL alignment. Think of it as hiring separate tutors for each subject, then having the student learn from all of them simultaneously — rather than one tutor trying to teach everything at once and confusing the student.

Several design choices deserve attention as a package: the Generative Reward Model collapses the actor-critic separation by making the model judge its own outputs — eliminating the need for separate scalar reward models and their costly human annotation. The three reasoning modes (Non-think / Think High / Think Max) are not just inference-time knobs — they are trained as distinct RL configurations with different length penalties, then unified through OPD. The FP4 QAT insight is particularly elegant: because FP8 has a larger dynamic range than FP4, the FP4→FP8 dequantization is lossless, meaning the entire existing FP8 training stack can be reused without modification. The full-vocabulary KL choice over token-level approximation trades compute for stability — a gamble that only pays off with the teacher scheduling infrastructure (hidden-state caching, sorted dispatch, async loading) described in 5.2.2.

The preemptible rollout service reveals a subtle correctness concern: length bias from preemption. If interrupted requests are regenerated from scratch, shorter sequences survive more often, implicitly rewarding brevity. The token-granular WAL is the fix — it preserves partial generation state so resumption is unbiased.

Why FP4→FP8 is lossless (the bit-level argument)

FP4 uses E2M1 format (2 exponent bits, 1 mantissa bit), giving it a very narrow dynamic range. FP8 E4M3 has 4 exponent bits — two more than FP4. When quantizing FP32 master weights, the FP4 sub-blocks (1×32 tiles) each get their own scale factor. When dequantizing to FP8 (128×128 blocks), as long as the ratio between the maximum and minimum FP4 sub-block scale factors within each FP8 block doesn't exceed the threshold that FP8's extra exponent bits can represent, all the fine-grained scale information is absorbed by FP8's wider dynamic range. The authors verify this empirically holds for their weights. This means the QAT forward pass is bit-identical to what a native FP8 training pipeline would produce — zero modification needed.

The length bias problem in preemptible rollout

Consider a batch of generation requests running when a preemption occurs. Shorter sequences are more likely to have already completed and been returned; longer sequences are more likely to be interrupted mid-generation. If you regenerate interrupted requests from scratch, you're implicitly creating a survival bias: the training data over-represents short outputs and under-represents long ones. Over many preemption events, the model learns to produce shorter responses. The token-granular WAL fixes this by persisting every generated token immediately, so resumption continues exactly where it left off — no bias toward any sequence length.
